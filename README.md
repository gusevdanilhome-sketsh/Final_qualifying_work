# Классификация дефектов микрополосковой линии

Данный программный комплекс предназначен для моделирования микрополосковой линии передачи с локальными неоднородностями (дефектами), генерации синтетических данных измерений и классификации типов дефектов с использованием методов машинного обучения (случайный лес). Проект включает в себя физическую модель линии, модель измерительной головки, квадратурного демодулятора, а также полный конвейер обработки данных: от генерации до обучения и оценки классификатора.

## Основные возможности

- Моделирование микрополосковой линии с учётом частотной дисперсии (эффективная диэлектрическая проницаемость, волновое сопротивление, постоянная распространения).
- Создание дефектов четырёх типов: утонение высоты полоска, утонение ширины полоска, утонение подложки, изменение диэлектрической проницаемости.
- Имитация измерительной системы с четырёхэлектродной головкой, формирователем суммарного и разностных каналов, квадратурным демодулятором (с возможностью добавления шума).
- Генерация синтетических данных для множества позиций сканирования и нескольких частот.
- Обучение классификатора случайного леса с подбором гиперпараметров (GridSearchCV, кросс-валидация).
- Визуализация данных: карта дефектов, годографы, PCA-проекции, частотные зависимости, матрица ошибок, важность признаков.
- Консольный интерфейс для управления всеми этапами работы.

## Конфигурация

Все параметры моделирования задаются в YAML-файле. Пример – `config/default.yaml`. Основные разделы:

- **line** – параметры микрополосковой линии (длина, ширина, высота подложки, толщина полоска, диэлектрическая проницаемость).
- **defects** – параметры дефектов: координаты центров, длина, типы (идентификаторы 1–4), множители для изменения геометрии/εr.
- **probe** – размер измерительной головки.
- **measurement** – частотный диапазон (ГГц), шаг, мощность падающей волны, отношение сигнал/шум (SNR).
- **scan** – параметры сканирования: отступ от края, шаг по координате, количество повторений в каждой позиции (для разных реализаций шума).
- **model** – параметры обучения: размер тестовой выборки, доля валидации, гиперпараметры для GridSearch, random_state.
- **logging** – уровень логирования и файл для логов.
- **parallel** – настройки многопроцессорной обработки (пока не реализована полностью).
- **paths** – пути для сохранения данных, модели, отчётов и графиков.

При необходимости можно создать несколько конфигурационных файлов для разных экспериментов.

## Использование

### Консольный интерфейс (CLI)

Для удобства предусмотрен интерактивный интерфейс, запускаемый командой:
```bash
python cli.py
```
При запуске можно указать путь к конфигурационному файлу (по умолчанию используется `config/default.yaml`). Меню предлагает следующие действия:

1. **Сгенерировать данные** – запускает `generate_data.py`, создаёт CSV-файл с признаками.
2. **Обучить модель** – запускает `train_model.py`, выполняет разделение данных, подбор гиперпараметров, обучение финальной модели на train+val, сохраняет модель и строит графики.
3. **Оценить модель** – запускает `evaluate.py`, загружает сохранённую модель и выводит метрики на всех данных (без разделения).
4. **Визуализировать данные** – запускает `visualize_data.py`, строит набор графиков по сгенерированным данным.
5. **Открыть папку с отчётами** – открывает директорию `reports/`.
6. **Открыть папку с графиками** – открывает `reports/figures/`.
7. **Изменить конфигурационный файл** – позволяет указать другой YAML-файл.
0. **Выход**

### Ручной запуск скриптов

Можно также запускать скрипты напрямую из командной строки:

**Генерация данных**:
```bash
python scripts/generate_data.py --config config/default.yaml
```

**Обучение модели**:
```bash
python scripts/train_model.py --config config/default.yaml
```

**Оценка модели**:
```bash
python scripts/evaluate.py --config config/default.yaml --model models/random_forest_model.pkl
```

**Визуализация данных**:
```bash
python scripts/visualize_data.py --config config/default.yaml
```

Для получения справки по аргументам используйте флаг `-h`.

### Рабочий процесс

1. Отредактируйте конфигурационный файл под свои нужды (параметры линии, дефектов, частоты, SNR).
2. Запустите генерацию данных (пункт 1 меню или `generate_data.py`). В папке `data/` появится CSV-файл.
3. Обучите модель (пункт 2). После завершения в `models/` будет сохранён файл `.pkl`, а в `reports/figures/` – графики (матрица ошибок, важность признаков, PCA). Текстовый отчёт сохраняется в `reports/classification_report.txt`.
4. При необходимости оцените модель на всех данных (пункт 3) – выводится accuracy, precision/recall/F1 по классам, матрица ошибок.
5. Для исследовательского анализа можно построить дополнительные визуализации (пункт 4): годографы, частотные зависимости, карту дефектов и др.

## Визуализация

Функции визуализации находятся в модуле `src.visualization`. Генерируются следующие графики:

- **Карта дефектов** – прямоугольники с указанием типов.
- **Годографы разностного канала Dx** – для выбранной частоты.
- **Многопанельный scatter I-Q** – для всех частот.
- **PCA-проекция** всех признаков на первые две главные компоненты.
- **Частотная зависимость средних значений каналов** (например, I_S) с доверительными интервалами.
- **Частотная зависимость фазы** разностного канала.
- **Важность признаков** (топ-N).
- **Фрагмент дерева решений** из случайного леса.
- **Матрица ошибок**.
- **Сравнительная карта** истинных и предсказанных классов для тестовых позиций.

Все графики сохраняются в `reports/figures/`.

## Модель и метрики

В качестве классификатора используется **RandomForestClassifier** из библиотеки scikit-learn. Гиперпараметры подбираются с помощью GridSearchCV на обучающей выборке. Оценка производится на тестовой выборке (30% от всех данных), внутри которой 50% отведено под валидацию (итого 70% train, 15% val, 15% test). Метрики: accuracy, precision, recall, F1-score (по классам и macro), матрица ошибок.

## Расширение и модификация

- **Добавление новых типов дефектов** – в `defect_t` можно определить собственные параметры и методы расчёта коэффициентов отражения/прохождения.
- **Изменение модели линии** – в `microstrip_line_t` можно заменить формулы эффективной проницаемости и волнового сопротивления.
- **Другие классификаторы** – в `classification.py` замените `RandomForestClassifier` на любой другой совместимый с scikit-learn.
