# Классификация дефектов микрополосковой линии

Проект предназначен для моделирования микрополосковой линии с дефектами, генерации синтетических данных измерений и классификации типов дефектов с помощью случайного леса.  
Включает в себя полный конвейер: создание модели линии и дефектов, симуляцию измерительной головки, формирование признаков (I/Q компоненты), обучение классификатора, визуализацию результатов и интерактивный CLI.

## Инструкция по созданию виртуального окружения Anaconda и запуску проекта

Данная инструкция поможет вам настроить изолированное окружение с помощью **Anaconda** и запустить программу.

### 1. Установка Anaconda
Если у вас ещё не установлена Anaconda, скачайте и установите её с [официального сайта](https://www.anaconda.com/products/individual).  
После установки убедитесь, что команда `conda` доступна в терминале (командной строке).  
Проверьте:  
```bash
conda --version
```

### 2. Клонирование или распаковка проекта
Поместите файлы проекта в удобную директорию. Например, создайте папку `microstrip_project` и скопируйте туда все предоставленные файлы (включая `cli.py`, `config/`, `src/`, `scripts/` и т.д.).  
Затем откройте терминал и перейдите в эту папку:
```bash
cd /путь/к/microstrip_project
```

### 3. Создание виртуального окружения
Создайте новое окружение с именем, например, `mpl_env`, и укажите версию Python (рекомендуется 3.9 или 3.10):
```bash
conda create -n mpl_env python=3.9
```
Подтвердите создание, нажав `y`.

### 4. Активация окружения
Активируйте созданное окружение:
```bash
conda activate mpl_env
```
В командной строке должно появиться `(mpl_env)`.

### 5. Установка зависимостей
Проект использует несколько библиотек. Основные зависимости перечислены в файле `requirements.txt`. Установите их с помощью `pip`:
```bash
pip install -r requirements.txt
```
Кроме того, в скрипте `cli.py` используется библиотека `colorama` для цветного вывода. Установите её дополнительно:
```bash
pip install colorama
```
*Примечание:* Если вы планируете запускать только отдельные скрипты (например, `generate_data.py`), `colorama` может не понадобиться, но для полноценной работы CLI она желательна.

### 6. Проверка структуры проекта
Убедитесь, что в корне проекта присутствуют:
- конфигурационный файл `config/default.yaml` (он уже есть в файлах);
- папки `src`, `scripts`, `data`, `models`, `reports` (последние три могут быть созданы автоматически при первом запуске).

### 7. Запуск программы
Для запуска основного интерфейса выполните:
```bash
python cli.py
```
Вы увидите главное меню с выбором режима (пользователь/администратор).  
При первом запуске рекомендуется сгенерировать данные (режим администратора → пункт 1), затем обучить модель (пункт 2) и изучить визуализацию.

### 8. Альтернативные способы запуска
Вы также можете вызывать отдельные скрипты из папки `scripts/` напрямую, указав путь к конфигу:
```bash
python scripts/generate_data.py --config config/default.yaml
python scripts/train_model.py --config config/default.yaml
python scripts/visualize_data.py --config config/default.yaml
```

### 9. Деактивация окружения
После завершения работы можно выйти из окружения командой:
```bash
conda deactivate
```

## Примечания
- Если у вас возникнут ошибки, связанные с отсутствием библиотек, убедитесь, что окружение активировано и все зависимости установлены.
- Для просмотра списка установленных пакетов в окружении выполните `conda list`.
- При использовании Windows может потребоваться предварительно выполнить `conda init` для корректной работы команды `conda activate`. Обычно это делается автоматически при установке.
